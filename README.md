# 🖥️ Stack vs Heap Memory Access Speed Analysis

## Computer Organization and Architecture (COA) Project

### 📖 Overview

This project analyzes the memory access behavior of **Stack Memory** and **Heap Memory** using the **Ripes RISC-V Simulator**. The study compares the execution characteristics of both memory regions by implementing RISC-V assembly programs and evaluating instruction count, clock cycles, and memory access patterns.

The project was developed as part of the **Computer Organization and Architecture (COA)** course in the B.Tech Computer Science and Engineering curriculum.

---

## 🎯 Objectives

* Understand Stack and Heap memory organization.
* Simulate memory access using RISC-V Assembly.
* Compare instruction count and execution cycles.
* Analyze memory access performance.
* Visualize processor execution using the Ripes Simulator.

## 📊 Key Findings

The experiment demonstrated the practical differences between stack and heap memory access. While stack memory requires stack pointer manipulation, heap (data) memory uses direct addressing through base registers. The project highlights how memory organization and addressing methods influence execution characteristics and overall program behavior.
